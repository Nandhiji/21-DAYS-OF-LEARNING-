An exception is an error that happens while the program is running.

So the program stops suddenly.

so the program gave a error , that scenario the exception handling will come . it will throw a arithimeticexception.

Why Exception Handling is Important

Without handling:

Program crashes ❌

With handling:

Program continues running ✅

Why Exception Handling is Important
Types of Common Exceptions:

Exception	                            Example
ArithmeticException	                       divide by zero
NullPointerException	                   using null object
ArrayIndexOutOfBoundsException	           accessing wrong array index
NumberFormatException                   	converting wrong string to number


This is how Java handles exceptions.

Example:

try{
   // risky code
}
catch(Exception e){
   // handle error
}

finally block:

Now we move to finally block.

Structure:

try{
   risky code
}
catch(Exception e){
   handle error
}
finally{
   always runs
}

The finally block runs whether exception occurs or not.
